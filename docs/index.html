Architecture
Five streams, one forecast

Why multiple sensing layers?

Peak electricity demand isn't driven by any single factor. It emerges from the interaction of extreme heat, population growth, EV adoption curves, datacenter buildout, and the built environment's thermal inertia.

The five-stream sensing layer architecture treats each domain as a separate token sequence. County embeddings are prepended as learned context vectors, allowing the model to capture county-specific demand signatures that a flat feature vector would miss.

Cross-stream attention then lets the model learn the interactions — how a heatwave's impact on demand differs in a county with high EV penetration versus one with expanding datacenter load.

Input streams

  • Weather + Time — Temperature, humidity, solar radiation, wind, temporal encoding
  • Extended Weather Patterns + Time — 3/7/14-day rolling aggregations capturing thermal accumulation
  • Geography / Demographics — Population density, urban fraction, elevation, climate zone
  • Heatwave Patterns — Duration, intensity, and nighttime recovery metrics
  • Planning for a sustainable future — Datacenter capacity, BEV adoption trajectories, grid infrastructure
